## 目标

## 已做决策
| 决策 | 理由 |
|------|------|
| 纯静态网页，零依赖 | 用户是代码小白，双击即用最简单 |
| 手动框选先于 AI | 快速跑通核心流程，AI 后期迭代 |
| 自写 GIF 编码器 | 避免 CDN 依赖，兼容 file:// 协议 |
| NES 56 色调色板 | 经典 8-bit 游戏配色，风格统一 |
| 像素尺寸可调 | 用户希望自由度高 |
| 先做 2 种动画 | 眨眼+呼吸，跑通后扩展 |

## 关键问题
1. GIF 编码器的 LZW 压缩实现复杂度 —— 需要仔细编写和测试
2. file:// 协议下部分浏览器可能限制 Canvas 的 toDataURL —— 需验证
3. 大图片在低配电脑上的像素化性能 —— 需控制图片尺寸上限
